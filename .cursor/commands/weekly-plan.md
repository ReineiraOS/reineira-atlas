# Weekly Review

Conduct the weekly sprint review and plan next week.

## Before Starting

Read these files:
- `.claude/docs/execution/SPRINT_LOG.md` — last week's progress
- `.claude/docs/intelligence/METRICS.md` — current numbers
- `.claude/docs/strategy/ROADMAP.md` — phase and priorities
- `.claude/data/decisions/` — recent decisions

## Process

1. For each priority in ROADMAP: assess status (done / on track / blocked / at risk)
2. Summarize what was accomplished this week
3. Flag blockers with proposed resolution
4. Define next week's priorities
5. **Append** new entry to top of SPRINT_LOG.md
6. Update ACTION_ITEMS.md if needed

## Acceptance Criteria

- [ ] Sprint log updated with this week's entry
- [ ] All priorities assessed
- [ ] Blockers flagged with proposed resolution
- [ ] Next week's priorities are clear

## Output

Return: weekly status report with progress, blockers, and next priorities.
